## 装饰器原理

装饰器是Python中一种强大的语法糖，本质是一个函数，其核心原理基于以下几点：

1. **闭包机制**：装饰器利用闭包来捕获外部函数的变量，从而在不修改原函数的情况下增强其功能
2. **高阶函数**：接受函数作为参数或返回函数的函数
3. **函数即对象**：Python中函数是一等公民，可以作为参数传递和返回

## 基础语法结构

```python
def decorator(func):
    def wrapper(*args, **kwargs):
        # 在调用原函数前执行
        result = func(*args, **kwargs)
        # 在调用原函数后执行
        return result
    return wrapper

@decorator
def target_function():
    pass
```

## 常见应用场景

1. **日志记录**：自动记录函数调用和参数
2. **性能计时**：测量函数执行时间
3. **权限验证**：检查用户权限
4. **缓存结果**：实现memoization
5. **路由装饰**：Web框架中的URL映射
6. **重试机制**：自动重试失败的操作
7. **输入校验**：验证函数参数类型

## 带参数的装饰器

```python
def repeat(times):
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for _ in range(times):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator

@repeat(3)
def greet(name):
    print(f"Hello, {name}!")
```

## 类装饰器

```python
class CountCalls:
    def __init__(self, func):
        functools.update_wrapper(self, func)
        self.func = func
        self.num_calls = 0
    
    def __call__(self, *args, **kwargs):
        self.num_calls += 1
        print(f"Called {self.num_calls} times")
        return self.func(*args, **kwargs)

@CountCalls
def example():
    pass
```

## functools.wraps 的重要性

必须使用 `@functools.wraps(func)` 装饰内部wrapper函数，以：
- 保留原函数的 `__name__` 和 `__doc__`
- 保留原函数的参数签名
- 使装饰后的函数在inspect模块中表现正常

## 装饰器执行顺序

多个装饰器的执行顺序从下到上：
```python
@decorator1
@decorator2
@decorator3
def func():
    pass
# 执行顺序: func -> decorator3 -> decorator2 -> decorator1
```

## 注意事项

1. 使用 `functools.wraps` 保留元数据
2. 使用 `*args, **kwargs` 处理任意参数
3. 注意装饰器对函数签名的改变
4. 考虑装饰器的可重用性和组合性

## 费曼学习法原理

费曼学习法基于"确认偏误"和"知识诅咒"两个认知心理学原理：

1. **确认偏误**：人们倾向于寻找支持自己已有观念的信息，而忽视矛盾的信息。通过主动暴露盲点，可以打破这种思维定式

2. **知识诅咒**：一旦掌握某项知识，就很难想象不知道它的状态。这导致专家难以用简单的语言向初学者解释复杂概念

费曼认为：真正理解一件事，就是能用简单的语言解释清楚。这种学习方式不仅巩固已有知识，更能发现理解中的漏洞和不足。

## 费曼学习法四步曲

### 第一步：选择概念

明确要学习的主题或概念。最好是具体且有一定复杂度的内容。写下所有已知信息，预设想要达成的理解程度和实际应用场景。

### 第二步：教授他人

这是关键步骤。想象向一个完全陌生的人解释，或实际向朋友讲解。要求：用最简单的语言，完整的逻辑，丰富的例子。写下来或说出来都可以，关键是清晰表达。

### 第三步：回顾卡点

在解释过程中，会发现说不清楚或逻辑跳跃的地方。这些就是知识盲点。回到原始材料重新学习这些部分，直到能够完整解释。

### 第四步：简化类比

用日常生活的比喻或类比来重新表述。如果能用身边的事物讲清楚，就说明真正掌握了。检验的标准是：小学生能听懂。

## 实操指南

1. **准备阶段**：选择一个核心概念，准备纸笔或空白文档，设定10-15分钟的专注时间

2. **讲解阶段**：在纸上写下"我今天要解释的概念是..."，然后用最简单的话写下解释

3. **识别问题**：大声朗读解释，注意卡顿和模糊的地方，标记所有"我不知道为什么"的语句

4. **填补漏洞**：查阅资料重新学习这些部分，再次尝试解释

5. **精炼表达**：用类比和比喻替换专业术语，确保5岁小孩也能理解

## 应用场景

- 学习新技术栈或框架
- 准备考试和面试
- 阅读技术文档后总结
- 向团队传授新知识
- 整理学习笔记

## 费曼学习法的优势

1. 主动学习，避免被动接受
2. 即时反馈学习效果
3. 强化知识记忆和内化
4. 培养系统化思维
5. 识别知识盲区

## 常见错误

1. 跳过"教授"环节，只看不写
2. 使用过多专业术语
3. 急于求成，没有深挖卡点
4. 类比不当，反而增加理解难度

## Markdown标题语法

Markdown支持两种标题语法：**ATX格式**和**SETEX格式**

### ATX格式（推荐）

使用井号 `#` 表示标题，一到六个井号对应一到六级标题：

```markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

### SETEX格式

使用等号和减号作为下划线：

```markdown
一级标题
============

二级标题
------------
```

## 层级结构规范

### 层级设计原则

1. **连续性**：层级应保持连续，避免跳级（如从H1直接跳到H3）
2. **唯一性**：同一文档中H1应唯一，通常作为文档主标题
3. **扁平化**：建议控制层级深度不超过4级，优先使用并列结构

### 推荐结构模板

```markdown
# 文档主标题

## 第一章

### 1.1 小节

#### 关键点

##### 补充说明
```

## 格式规范

### 标题行内要求

1. **空格规范**：井号后应加空格，如 `# 标题` 而非 `#标题`
2. **末尾处理**：ATX格式的井号后不应加内容，自动生成ID会受影响
3. **标题字符**：避免在标题中使用特殊字符，必要时使用转义

### 视觉一致性

1. **同级对齐**：同级标题应保持视觉上的对齐
2. **间距统一**：标题与上下内容的间距建议保持一致
3. **长度控制**：单行标题不超过80字符，便于阅读和SEO

## 最佳实践

### 文档结构设计

```markdown
# 项目文档

## 概述
简要介绍项目背景和目的

## 功能特性
### 核心功能
详细说明核心功能
### 扩展功能
说明扩展能力

## 使用指南
### 环境配置
### 快速开始
### 进阶用法

## 常见问题
## 更新日志
## 附录
```

### 命名规范

1. **语义化**：标题应准确反映内容主题
2. **简洁性**：避免冗长标题，使用精炼词汇
3. **可检索**：使用关键词便于搜索定位
4. **大小写**：英文标题推荐首字母大写或全小写，保持一致

### SEO优化

1. 核心关键词尽量出现在H1中
2. H2用于主要章节主题
3. H3-H4用于具体子主题
4. 避免标题堆砌关键词

## 工具支持

### 渲染器兼容性

### 自动生成目录

大多数Markdown工具支持自动从标题生成目录，依赖正确的标题层级结构。

## 常见错误

1. ❌ 使用错误数量的井号（超过6个）
2. ❌ 标题与正文之间缺少空行
3. ❌ 同一层级使用不同语法
4. ❌ 标题中使用过多强调标记
5. ❌ 层级混乱导致目录生成错误

## Linux.do 社区入门指南

### 社区简介

Linux.do（简称 L站）是一个成立于 2024 年 1 月 17 日的国内技术社区，由创始人 **pengzhile**（又称秦始皇、始皇）创建。社区基于 Discourse 平台搭建，用户们自称「兵马俑」，以真诚、友善、团结、专业为社区精神。

### 注册流程

L站采用**邀请制注册**，新用户无法直接申请加入。具体流程如下：

1. **获取邀请码**：通过朋友邀请、海鲜市场购买或与其他用户交易获得
2. **完成注册**：使用邀请码注册账号
3. **撰写小作文**：提交个人介绍供管理员审核
4. **等待准入**：审核通过后收到准入邮件，正式成为兵马俑

### 等级系统

社区采用 Discourse 信任度等级系统：

- **等级 0**：新用户，完成注册后初始等级
- **等级 1**：需进入 5 个话题、阅读 30 篇帖子、花费 10 分钟
- **等级 3**：可生成邀请码，邀请新用户加入
